### Vulnerability: OS Command Injection via Unsanitized Environment Variable

The binary performs the following operations (confirmed via `strings` and `ltrace`):

1. **`geteuid()` / `getegid()`** — Gets the effective UID/GID (flag02)
2. **`setresuid()` / `setresgid()`** — Sets all UIDs/GIDs to flag02 (the SUID owner)
3. **`getenv("USER")`** — Reads the `USER` environment variable (user-controlled!)
4. **`asprintf(&buf, "/bin/echo %s is cool", user)`** — Formats the USER value directly into a command string
5. **`system(buf)`** — Passes the unsanitized string to `/bin/sh -c` for execution

### The Flaw
The `USER` environment variable is interpolated directly into a shell command passed to `system()` **without any input validation or sanitization**. Since environment variables are fully user-controlled, an attacker can inject arbitrary shell commands using metacharacters like `;`, `$()`, `|`, or backticks.


1. `/bin/echo` (prints empty line)
2. `getflag` (our injected command — runs as flag02!)
3. `is cool` (fails — harmless)

1. `sub $0x60, %esp` - Allocates 0x60 (96) bytes on the stack
2. `getenv("GREENIE")` - Gets the GREENIE environment variable
3. If NULL, calls `errx(1, "please set the GREENIE environment variable")`
4. Stores 0 at `esp+0x58` - This is the target variable to overwrite
5. `strcpy(esp+0x18, getenv_result)` - Copies GREENIE into buffer at esp+0x18
6. Compares `esp+0x58` with `0x0d0a0d0a`
7. If equal: prints "you have correctly modified the variable"
8. If not: prints "Try again, you got 0x%08x"

So the buffer starts at esp+0x18 and the variable is at esp+0x58.
The offset is 0x58 - 0x18 = 0x40 = 64 bytes.

We need to overflow the buffer with 64 bytes of padding + the value 0x0d0a0d0a (in little-endian).

0x0d0a0d0a in little-endian is: \x0a\x0d\x0a\x0d

Wait, 0x0d0a0d0a:
- Byte 0 (least significant): 0x0a
- Byte 1: 0x0d
- Byte 2: 0x0a
- Byte 3 (most significant): 0x0d

So in little-endian memory: 0a 0d 0a 0d

The challenge is that 0x0a is newline and 0x0d is carriage return. Since this comes from an environment variable (not stdin), we can use Python to set the env var with these bytes.
